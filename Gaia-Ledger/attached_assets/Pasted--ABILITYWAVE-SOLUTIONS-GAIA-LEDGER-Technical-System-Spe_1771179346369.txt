Each CropSheet must:

Have independent historical aggregation

Generate its own graph instance

Persist data independently

Not share state with other crop sheets

Backend Requirement

Endpoint:

GET /api/crop-sheets/:id/health-history


Query:

Filter by cropSheetId

Group by date

Return aggregated health counts

Frontend Requirement

Each CropSheet page must:

Render dynamic graph

Allow metric selection (Health Status count, leaf count, etc.)

Allow filtering by date range

B. Dynamic Graph Creation Per Crop Sheet

When:

A new CropSheet is created

System must:

Automatically enable graph tracking

No shared graph instance

Separate dataset per sheet

Graph component must accept:

cropSheetId as prop


Never use global graph state.

Users must be able to:

Update color field in database
